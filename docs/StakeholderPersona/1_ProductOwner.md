# 페르소나 1: Product Owner (서비스 오너)

## 👑 프로필: 사라 (Sarah)
*   **나이 / 직업:** 35세 / B2B SaaS 프로덕트 매니저 (Lead PM)
*   **성향:** 수치 기반의 데이터 드리븐(Data-driven) 의사결정을 선호. 비즈니스 가치(사용자 리텐션, 활성도)에 집착함.
*   **기술적 이해도:** 중간. 엔지니어와 대화는 통하지만 코드를 직접 짜지는 않음.

## 🎯 핵심 목표 (Goals)
*   알렉스(소비자)가 이 서비스를 하루 만에 지우지 않고 **내일 다시 들어오게(Day 1 Retention)** 만드는 것.
*   단순한 개인용 장난감에 머물지 않고, 다른 주제(예: 주식, 블록체인 등)로 쉽게 피보팅(Pivoting)할 수 있는 서비스 모델을 정립하는 것.

## 🤬 주요 페인포인트 (Pain Points)
*   "개발팀이 기술적인 완벽(예: GraphQL 도입 등)에 집착하느라 정작 사용자가 원하는 '예쁜 공유 버튼' 같은 핵심 피쳐 개발을 뒤로 미루는 것."
*   "토요일 오전에 발행되는 주간(Weekly) 편지가 데일리 뉴스랑 똑같이 생겨서 사용자가 특별함을 못 느끼는 것."

## 🤝 AI News Archive 시스템에 바라는 점 (Needs & Expectations)
*   **비주얼 계층화:** 주간 베스트(Weekly Top 3) 뉴스는 누가 봐도 '이 주에 제일 중요했던 기사'라는 위엄(테두리, 순위 뱃지)이 있어야 한다.
*   **바이럴 루프 (Viral Loop):** 각 뉴스 카드에 '공유하기' 버튼이 명확히 있어야 하며, 카톡으로 링크를 보냈을 때 썸네일과 제목이 예쁘게 잡혀야(OG Meta) 한다.
*   **Empty State 방어:** 데이터가 없을 때 "404 Not Found" 같은 무서운 에러 대신, 위트 있는 일러스트가 떠야 한다.

## 💡 프로젝트 영향력 (Impact on Project)
*   사라의 입김으로 인해 백엔드가 주간 빈도수를 연산하는 로직을 짰으며, 프론트엔드 작업 목록에 **프리미엄 UI 카드 제작**, **OG Meta 태그 적용**, **우아한 Empty State 노출** 기능이 강제로 삽입되었습니다.
