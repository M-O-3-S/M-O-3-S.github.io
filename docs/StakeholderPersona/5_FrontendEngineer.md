# 페르소나 5: Frontend Engineer (프론트엔드)

## 💻 프로필: 리암 (Liam)
*   **나이 / 직업:** 28세 / 프론트엔드 (웹) 개발자
*   **성향:** 화려한 프레임워크(React, Vue 등)보다 가볍고 빠른 'Vanilla JS + HTML + CSS' 조합의 극한 성능(Performance) 최적화에 쾌감을 느낌.
*   **기술적 이해도:** 아주 높음. DOM 조작 및 브라우저의 비동기 네트워크 동작 원리, Static Hosting의 제약 사항을 꿰뚫고 있음.

## 🎯 핵심 목표 (Goals)
*   데이터베이스(서버)가 없고 단순히 GitHub Pages에 올려진 정적(Static) 파일만으로도, 마치 '살아있는 웹 애플리케이션(SPA)'처럼 빠르고 번쩍이는 체감 속도를 흉내 내는 것.
*   1년 치 JSON 텍스트 파싱 때문에 사용자 브라우저 탭(메모리)이 뻗거나 버벅거리는 현상을 원천 차단하는 것.

## 🤬 주요 페인포인트 (Pain Points)
*   "기획자가 '작년 데이터에서 검색하게 해주세요'라고 하는데, 우리 서비스는 구버전 정적 호스팅이라 100MB짜리 통 JSON을 프론트가 브라우저에서 뒤져서 렌더링해야 하는 악몽."
*   "비동기로 데이터를 떠오느라 화면이 0.5초 동안 새하얗게 비어있어 앱이 죽은 것처럼 보일 때 느끼는 강박증."
*   "텔레그램 딥링크(`#tagName`이나 `#date`)를 눌러서 들어왔을 때, 바닐라 JS 로딩이 완료되지 못해 화면 최상단에서 멈춰버리는 이슈."

## 🤝 AI News Archive 시스템에 바라는 점 (Needs & Expectations)
*   **파일 파티셔닝(Partitioning):** 백엔드에게 무조건 날짜 단위(`daily.json`), 태그 단위(`tags.json`)로 통파일을 잘게 분할해서 넘겨달라고 읍소 (클라이언트는 필요한 10KB만 Fetch해야 함).
*   **Skeleton Loading:** 데이터를 받아오기 전까지 사용자에게 보여줄 '회색빛 뼈대(Skeleton Placeholder)' UI 시안이 필수.
*   **ScrollIntoView 제어권:** 앵커 태그가 적용되려면 화면 DOM이 렌더된 시점에 JS가 직접 타겟으로 강제 스크롤시켜야 하므로 이를 요구사항에 명시.

## 💡 프로젝트 영향력 (Impact on Project)
*   리암의 절규와 기술적 한계 설정으로 인하여 백엔드는 **분할된 JSON 굽기 구조**와 **태그 JSON 최대 30일 제한 렌더링**을 수용하게 되었고, **스켈레톤 UI와 Empty State 화면 도입**이 UX 가이드라인에 필수 항목으로 지정되었습니다.
