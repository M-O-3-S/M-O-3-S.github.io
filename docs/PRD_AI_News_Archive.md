# 나만의 AI 뉴스 아카이브

---

## 1. 서비스 개요

AI는 매일 새로운 기술과 소식이 쏟아지는 분야다. 관심은 있지만 방대한 정보를 매일 직접 찾아보기란 쉽지 않다.
이 시스템은 국내외 AI 관련 뉴스를 매일 자동으로 모아서 읽기 쉽게 정리해주는 나만의 AI 뉴스 아카이브다.
기술 동향뿐 아니라 AI가 우리 사회와 일상에 미치는 변화까지 함께 살펴볼 수 있으며, 쌓인 뉴스를 언제든 되돌아보며 흐름을 파악하는 것을 목적으로 한다.
나아가 AI 뉴스 외에도 관심 있는 다른 주제로 확장하여 활용할 수 있도록 설계한다.

---

## 2. 뉴스 카테고리

Daily News 및 Weekly News 공통으로 적용되는 뉴스 카테고리는 다음과 같다.

| 카테고리 (태그용 최적화 명칭) | 설명 |
|----------|------|
| 엣지/임베디드 AI | 디바이스 내 AI 추론, 경량 모델, NPU, MCU 등 |
| AI 개발도구 | IDE, 프레임워크, 소프트웨어 엔지니어링 도구 및 코딩 |
| 거대 언어 모델 (LLM) | 대형 언어모델 개발·출시·벤치마크 및 NLP 기술 |
| AI 하드웨어/반도체 | M5, 서버 인프라, 실리콘 및 하드웨어 칩 등 물리적 엔지니어링 |
| 로보틱스/자율주행 | 물리적 AI(Physical AI), 자율주행, 휴머노이드 동향 |
| AI 비즈니스/트렌드 | 기업 전략, 비즈니스 투자, IT 비즈니스 분석 및 AI 사회 이슈 통합 |
| AI 연구/논문 | arXiv 논문 발췌, 최신 논문 발표 요약 |

## 3. 뉴스 수집

### 3-1. 수집 소스
- 뉴스 정보는 사전에 등록된 RSS Feed 목록으로부터 수집한다.

### 3-2. 수집 주기
- 한국시간(KST) 기준 매일 오전 7시에 자동 수집한다.

### 3-3. 중복 제거
- 기본적으로 URL 해시를 비교하여 동일한 기사가 여러 출처에서 중복으로 수집되는 경우 한 건만 저장한다.
- 서로 다른 URL이더라도, 정규화된(Normalized) 기사 제목과 발행일자가 동일한 경우 같은 기사로 간주하고 중복 처리한다.
- 이미 수집되어 처리된 기사는 다시 처리하지 않는다.

### 3-4. 최초 실행 및 데이터 수집
- 시스템 최초 구동 시 (DB가 비어있을 경우), 뉴스 소스에서 제공하는 과거 데이터(최대 N일치)를 일괄 수집하여 빈 아카이브를 방지한다.

## 4. 데이터 처리 및 AI 분석

### 4-1. 처리 모델
- 생성형 AI를 사용하여 수집된 뉴스 데이터를 처리 및 분석한다.
- **[Agnostic (모델 교체 용이성) 설계]:** 초기에는 `gemma3:4b` 모델을 기준으로 설계하되, 향후 퀄리티 향상을 위해 `Llama3.2 8B`나 `Gemma3 9B` 등으로 모델명(파라미터)만 변경하면 즉각 교체(Scale-up)가 가능한 '모델-애그노스틱(Model-Agnostic)' 구조로 코드를 작성해야 한다. 프롬프트는 특정 모델에 종속되지 않는 범용 형태를 유지한다.

### 4-2. 처리 항목

| 항목 | 처리 방식 | 비고 |
|------|-----------|------|
| 카테고리 분류 | 카테고리 선택지를 명시한 프롬프트로 분류 | 선택형 지시로 정확도 향상 |
| 뉴스 요약 | 최대 300자 이내 한국어 요약 생성 | |
| 태그 생성 | 제목 + 요약 분석 후 자동 생성 | 아래 태그 규칙 참고 |

### 4-3. 태그 생성 규칙 및 예외 처리
- 개별 뉴스의 태그는 반드시 1개 이상 3개 이하로 생성한다.
- 태그 내 띄어쓰기는 허용하지 않는다.
- 의미가 유사하거나 중복되는 태그는 생성하지 않는다. (예: `LLM`과 `거대언어모델`을 동시에 생성하지 않음)
- 글자 길이 제한:
  - 한글: 최소 1자, 최대 10자
  - 영문: 최소 2자, 최대 20자 (단어 간 구분이 필요한 경우 각 단어의 첫 글자를 대문자로 표기, 예: `GenerativeAI`)
- **[예외 처리]** AI 모델이 규칙(300자 초과, 태그 4개 이상 생성 등)을 어긴 경우 시스템은 1회 재시도(Retry)하며, 재시도 후에도 실패할 경우 기본 카테고리를 태그 1개로 임시 할당하여 저장한다.
- 영문 뉴스 기사가 수집된 경우, 프롬프트를 통해 반드시 **한국어로 요약**을 생성하도록 강제한다.

## 5. Daily News

### 5-1. 개요
최근 24시간 이내 수집된 AI 뉴스를 카테고리별로 분류하여 제공한다.

### 5-2. 발행 제목
```
AI News - {YYYY-MM-DD}
```

### 5-3. 개별 뉴스 구성

```
제목  : {뉴스 제목}
요약  : {최대 300자 가량의 뉴스 요약}
Tag   : #{Tag1} [, #{Tag2} [, #{Tag3}]]  ← 1개 이상 3개 이하
URL   : {뉴스 전체 기사가 있는 URL Link}
```

### 5-4. 발행 채널
- **Telegram**: 수집 완료 후 자동 발송
- **GitHub Pages**: `http://m-o-3-s.github.io` 에 정적 페이지로 게시

## 6. Weekly News

### 6-1. 개요
한 주간 수집된 뉴스 중 카테고리별 주요 뉴스를 선별하여 제공한다.

### 6-2. 발행 시각 및 집계 기준
- 한국시간(KST) 기준 매주 **토요일 오전 8시**에 발행한다.
- 집계 기준: **지난주 토요일 발생분부터 금요일 발생분(이번주 토요일 오전 7시 수집 분)**까지, 최근 7일간 수집된 뉴스를 대상으로 한다.

### 6-3. 뉴스 선정 기준
- 카테고리별로 해당 주기 동안 **가장 많이 등장한 태그**를 집계한다.
- 태그 빈도가 높은 순으로 뉴스를 정렬하여 카테고리당 **최대 3건**을 선정한다.
- 동점인 경우 최신 기사를 우선한다.

### 6-4. 개별 뉴스 구성
기본적인 항목(제목, 요약, 태그, URL)은 Daily News와 동일하지만, UX를 고려하여 데일리 모델과 확연히 구분되는 **시각적 '프리미엄' UI (순위 뱃지 표기, 테두리 강조 등)** 를 적용하여 차별화한다.

### 6-5. 발행 채널
Daily News와 동일하게 Telegram 및 GitHub Pages를 통해 발행한다.

## 7. 웹 인터페이스 (GitHub Pages)

### 7-1. 지원 디바이스 (Responsive Web)
- PC(데스크탑) 브라우저와 모바일 브라우저 환경을 모두 지원하는 **반응형 웹(Responsive Web)** 으로 구현한다.
- 데스크탑에서는 넒은 화면을 활용한 투 컬럼(Two-column) 레이아웃을, 모바일에서는 상하 스크롤 위주의 싱글 컬럼 레이아웃을 제공한다.

### 7-2. 메인 페이지 (Daily News & Weekly Best)
- 상단 네비게이션/탭을 통해 **데일리 뉴스**와 **주간 베스트(Weekly Top)** 화면을 직관적으로 전환할 수 있게 한다.
- **[테마 토글 지원]** 사용자의 작업 환경(어두운 개발 환경 vs 밝은 사무실 환경)에 맞춰 언제든 **다크 모드와 라이트 모드를 자유롭게 전환할 수 있는 테마 버튼**을 상단에 제공하며, 접속 시 OS 기본 설정(`prefers-color-scheme`)을 우선 적용한다.
- 가장 최근 Daily News를 기본 메인 뷰로 표시한다.
- 카테고리별 탭 또는 섹션으로 구분하여 보여준다.
- 피로도를 줄이기 위해 기사 목록은 카드 형태 또는 요약문 아코디언(펼침/접힘) UI를 제공하여 한눈에 훑어보기 편하게 구성한다.
- **[공유 기능 지원]** 각 뉴스 카드 하단에 'URL 복사' 방식의 공유 아이콘을 제공하며, 시스템 전체적으로 외부 카톡 등 공유 시 내용이 보이도록 OG Meta tags를 지원한다.
- 수집된 뉴스가 없거나 로딩 중인 상태(Empty/Loading State)에 대해 딱딱한 에러 화면 대신 친절한 안내 문구와 부드러운 애니메이션(Shimmer 등)을 제공한다.

### 7-3. 데일리 아카이브(타임머신) 조회
- 좌측 사이드바의 **날짜 선택(캘린더 위젯)**과 "최근 3일", "지난주" 등의 퀵 필터를 제공하여 과거 날짜의 Daily News를 쉽게 조회할 수 있게 한다.
- **아카이브 보관(조회) 가능 기간:** GitHub Pages의 정적 스토리지 최적화를 고려하여, 캘린더를 통한 데일리 뉴스 과거 조회는 **최근 30일(약 1개월)** 치만 제공하는 것을 기본 스펙으로 한다. (이전/다음 달 화살표 `< >`로 월 단위 이동 가능)

### 7-4. 위클리 아카이브(과거 주간 베스트) 조회
- 'Weekly Top' 탭 진입 시 기본적으로 가장 최근 발행된 주차의 베스트 뉴스를 표출한다.
- 리스트 상단 내비게이션 바에 `< 이전 주 (Week 9) | 다음 주 >` 형태의 페이지네이션 버튼을 제공하여 과거 주차별 위클리 뉴스를 순차적으로 열람할 수 있도록 설계한다.

### 7-5. 태그 검색 생태계
- 기사 목록은 수집일 기준 최신순으로 표시된다.
- 뉴스에 표시된 개별 태그를 클릭하면 해당 태그 목록 페이지로 이동한다.
- 태그 무한 증식으로 인한 복잡도를 줄이기 위해, "최근 한 달간 인기 태그 Top 20" 화면을 제공하여 사용자의 탐색을 돕는다.
- 아카이브 시스템 내에서 다른 뉴스로 이동하거나 원문을 조회할 때 동선이 매끄럽도록, 새 창 띄우기 원칙을 준수한다.

## 8. Telegram 알림

Telegram 알림은 뉴스가 발행되었음을 사용자에게 알리는 것을 목적으로 한다.
발행된 뉴스의 상세 내용은 GitHub Pages 웹사이트에서 확인한다.

### 8-1. Daily 알림
- 매일 뉴스 수집 및 처리 완료 후 자동 발송한다.
- 알림 메시지 상단에 오늘 날짜와 GitHub Pages 링크를 포함하며, 사용자의 흥미 유발을 위해 **오늘의 주요 뉴스 제목 1~3개(Bullet point 형태)**를 미리보기로 함께 발송한다.

### 8-2. Weekly 알림
- 매주 토요일 오전 8시에 주간 뉴스가 발행되었음을 알리는 메시지를 발송한다.
- 알림 메시지에는 해당 주차 정보와 GitHub Pages 링크를 포함한다.
