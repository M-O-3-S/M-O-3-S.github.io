# UX 및 시스템 UI 가이드라인 (Multi-Persona Design)

AI News Archive의 정적 프론트엔드(GitHub Pages) 레벨에서의 사용자 경험과 UI 정책을 4개의 디자인 전문 페르소나 관점에서 정의한 가이드라인입니다.

---

## 1. 🎨 UI/Visual Designer 관점 (테마 및 컴포넌트)
**목표:** 개발자 친화적이고 눈이 편안한 시각적 계층 구조 설계

*   **Color Palette (OS 동기화 및 라이트/다크 양방향 지원)**
    *   개발자(다크 선호)와 사무직 홍보 담당자(라이트 선호) 모두를 만족시키기 위해 **운영체제 기본 설정(`prefers-color-scheme`)을 우선적으로 따르며, 헤더에 토글 버튼을 제공**합니다.
    *   **Dark Mode (개발자 타겟):** 짙은 네이비 또는 차콜 (`#121826` 등) 계열로 눈의 피로를 최소화.
    *   **Light Mode (사무직 타겟 신규):** 사무실 모니터나 사내망 공유 시 위화감이 없는 깔끔한 흰색/밝은 회색 (`#F8FAFC`, `#FFFFFF`) 배경과 명확한 검정/진회색 텍스트.
    *   **Primary Accent:** 링크 및 강조 요소에 신뢰감을 주는 블루 계열 (`#3B82F6` 등)이나 AI 느낌을 주는 보라 계열 (`#8B5CF6`) 사용.
    *   **Text:** `rgba(255,255,255, 0.9)` 의 Primary 텍스트와 `0.6` 수준의 Secondary 텍스트 구분.
*   **Typography**
    *   가독성이 높은 산세리프 폰트(Pretendard, Inter 등) 적용.
    *   **제목(Title):** 요약 본문과 명확히 구분되도록 폰트 웨이트(Bold)와 크기 차이 배치.
*   **Component: 카드(Card) & 아코디언(Accordion)**
    *   각 기사는 독립된 모듈인 **'카드'**로 묶여야 하며, 초기 진입 시 카드는 접힌 상태(Collapsed)여야 합니다.
    *   카드를 탭/클릭하면 요약문이 아래로 스무스하게 펼쳐지도록(Accordion Expand) 설계.

## 2. ♿ Accessibility (접근성 및 가독성) 관점
**목표:** 다양한 환경(모바일/데스크탑)에서 방해 없는 읽기 경험 제공

*   **여백(Spacing)과 터치 타겟:**
    *   모바일 환경(텔레그램에서 유입)을 최우선으로 고려(Mobile First)하여 카드 간 여백(margin)과 내부 여백(padding)을 충분히 확보. 넓은 여백으로 콘텐츠 간의 간섭 최소화.
    *   버튼(예: 원문 보기, 태그 클릭)은 모바일 환경에서 오터치가 발생하지 않도록 최소 44x44px 이상의 터치 영역 확보.
*   **요약문 줄바꿈 및 글자수:**
    *   최대 300자로 생성될 요약문 공간의 줄바꿈(Line-height)은 `1.6` 이상으로 설정하여 가독성 증대.

## 3. 🔍 UX Researcher 관점 (정보 구조 및 탐색 플로우)
**목표:** 매일 반복되는 방문 경험의 피로도 단축과 효율적 탐색

*   **PC/모바일 반응형 레이아웃 구조 (Responsive Layout):**
    *   **모바일 (Mobile):** 상하 스크롤 중심의 1단(Single Column) 레이아웃. 상단에 네비게이션과 태그 칩이 위치하고 아래로 뉴스 카드가 이어집니다.
    *   **데스크탑 (PC):** 넓은 화면을 활용한 2단(Two Column) 레이아웃. 좌측 사이드바에 브랜드 심볼, 날짜 달력, 인기 태그 클라우드를 고정(Sticky)시키고, 우측 메인 영역에서 넓은 아코디언 카드를 리스트업하여 스크롤 피로도를 줄입니다.
*   **상단 네비게이션 구조 (Header Layout - Mobile 기준):**
    *   좌측 상단: 서비스 로고 "AI News Archive"
    *   우측 상단: '날짜 선택(Date Picker)' 및 '주간 베스트(Weekly Top)' 퀵 링크 햄버거 메뉴 처리.
*   **태그 생태계 필터 (Top 20 Tags):**
    *   사용자가 특정 태그 칩을 클릭하면, 화면 전환이나 스크롤 이동 없이 해당 카테고리로 필터링 된 화면이 즉시(Instant) 보이는 플로우 제공.
*   **텔레그램 진입 시나리오 (Telegram Handoff):**
    *   모바일 텔레그램 알림 링크를 타고 올 때, '오늘의 뉴스' 혹은 '해당 뉴스 카드의 앵커(#)' 위치로 바로 렌더링되게 하여 탐색 Depth 최소화.

## 4. 🔗 Frontend Web Designer 관점
**목표:** 정적 사이트(Static) 한계 내에서의 성능 고려 및 상태(State) 피드백

*   **로딩 상태 (Loading State) 피드백 (비기술 직군 배려):**
    *   단순한 회색 네모 깜빡임은 '오류'나 '해킹'으로 오해받을 수 있으므로, 데이터 Fetch 시 **부드럽게 빛이 흘러가는 쉬머(Shimmer) 애니메이션 스켈레톤**을 적용합니다.
    *   초기 데이터 진입 시에는 "최신 AI 뉴스를 불러오는 중입니다..." 등 **상황을 설명하는 친절한 텍스트 캡션이나 부드러운 스피너**를 스켈레톤과 함께 배치하여 사용자를 안심시킵니다.
*   **인터랙션(Hover / Active) 피드백:**
    *   마우스 커서나 터치 시, 해당 카드나 태그 버튼의 배경이 살짝 밝아지거나, '원문 보기' 링크에서 언더라인이 생기는 등 즉각적인 반응성 부여.
    *   모바일에서의 접힘/펼침 동작 시 애니메이션 적용(`transition: max-height 0.3s ease`).

## 5. 🏃‍♂️ 핵심 사용자 시나리오 (User Scenarios & UI Flow)
**목표:** 주요 사용 패턴에 따른 화면 간 이동 및 상호작용(Interaction) 정의

### 🟢 시나리오 1: 데일리 루틴 (아침 뉴스 출근길/업무시작 소비)
*   **상황:** 아침 7시 5분 이후, 수집된 주요 뉴스를 가볍게 훑어보는 가장 빈번한 패턴
*   **[모바일 Flow] 텔레그램 연동 출근길 리딩:**
    1.  **[텔레그램]** "오늘의 주요 뉴스 미리보기" 확인 후 발송된 링크 터치
    2.  **[메인 화면]** 오늘자 'Daily News' 화면 즉시 렌더링
    3.  **[탐색 및 소비]** 스크롤하며 제목 스캐닝 👉 아코디언 카드 터치하여 요약 읽기 👉 창 닫기
*   **[PC Flow] 북마크 접속 업무시작 리딩:**
    1.  **[웹 브라우저]** 출근 후 PC 브라우저 북마크로 아카이브 메인 페이지 접속
    2.  **[탐색 및 소비]** 넓은 투-컬럼 레이아웃의 우측 기사 리스트를 통해 빠르게 소비

### 🔵 시나리오 2: 딥다이브 (특정 기술 스택 트렌드 몰아보기)
*   **상황:** 특정 기술 키워드(예: #EdgeAI, #LLM)의 트렌드를 연속으로 확인하고 싶음
*   **[PC Flow] 데스크탑 사이드바 적극 활용:**
    1.  **[메인 화면]** 투-컬럼의 좌측 고정(Sticky) 사이드바 위치 확인
    2.  **[태그 클릭]** 인기 태그 Top 20 칩 중 원하는 카테고리(예: `#EdgeAI`) 클릭
    3.  **[목록 렌더링]** 우측 리스트 영역만 화면 전환 없이 필터링된 기사로 교체됨 (몰아보기)
*   **[모바일 Flow] 가로 스와이프 태그 칩 활용:**
    1.  **[메인 화면]** 상단 리스트 위쪽에 위치한 태그 칩(Chip) 컴포넌트들을 가로 스와이프
    2.  **[태그 터치]** 원하는 태그 터치 시, 하단 카드 리스트 UI 즉각 갱신

### 🟡 시나리오 3: 타임머신 (과거 데일리 날짜 및 위클리 주차 조회)
*   **상황:** 지난주 수요일 등 특정 데일리 날짜로 돌아가거나, 지지난주의 '주간 베스트' 뉴스를 확인하고 싶음.
*   **[모바일 Flow] 팝업 캘린더 호출 (Daily):**
    1.  **[헤더 아이콘]** 데일리 탭에서 상단 네비게이션 바의 '달력(Calendar)' 모양 아이콘 탭
    2.  **[날짜 픽커]** 나타난 글래스모피즘 팝업 달력(이전 달 `< >` 이동 지원, 최대 30일)에서 원하는 날짜 터치
    3.  **[결과 렌더링]** 과거 Daily News 뷰로 하단 카드 리스트 갱신
*   **[PC Flow] 사이드바 캘린더 위젯 활용 (Daily):**
    1.  **[위젯 조작]** 좌측 사이드바 영역에 포함된 전체 한 달(최대 31일 표출) 미니 달력에서 이전 달 화살표 `<` 또는 과거 날짜 직접 클릭
    2.  **[결과 렌더링]** 우측 메인 영역 제목에 `Latest AI News ({Date})` 표시와 함께 과거 기사 표출
*   **[PC/모바일 Flow] 과거 주간 베스트 조회 (Weekly):**
    1.  **[위클리 탭 진입]** 상단 탭에서 'Weekly Top' 선택
    2.  **[주차 이동 네비게이션]** 아티클 목록 상단에 위치한 `< 이전 주 (Week 9)` 버튼 터치
    3.  **[결과 렌더링]** 과거 주차의 '주간 베스트 3위' 화면으로 부드럽게 전환

## 6. 🚨 예외 상태 및 추가 구현 요구사항 (Edge Cases)
**PO 및 프론트엔드 관점의 누락 방지 핵심 UI/UX 검토 항목**

*   **🏆 주간 베스트 (Weekly Top 3) 뷰 (PO 관점):**
    *   데일리 뉴스 카드와 확연히 다른 '프리미엄' 시각적 처리가 필요함.
    *   글래스모피즘 기반의 네온(Neon) 테두리 하이라이트 및 `[Top 1 🔥]` 등의 순위 뱃지를 명확히 삽입하여 시각적 위계를 높임.
*   **⏳ 로딩 스켈레톤 (Skeleton UI) 상태 (FE & UX 통합 관점):**
    *   정적 페이지에서 외부 JSON 데이터를 `fetch()`로 가져올 때 발생하는 지연 시간 동안 빈 화면이 보이지 않도록 함.
    *   에러(서버 다운)처럼 보이지 않도록, 콘텐츠(아코디언 형태) 본뜬 블록 위에 **쉬머(Shimmer) 애니메이션 처리**를 하고 상단에 **"데이터를 불러오고 있습니다"라는 명시적 안내 문구를 병기**하여 불안감을 억제.
*   **📭 엠티 (Empty) 상태 화면 (PO & FE 관점):**
    *   "최근 한 달 인기 태그"에 진입했는데 해당 태그 기사가 지워졌거나, 수집 오류로 과거 특정 날짜에 데이터가 아예 없는 경우.
    *   차갑고 기계적인 에러 메시지가 아닌, 서비스 테마에 맞는 은은한 기하학적 일러스트와 함께 "해당 일자/태그에는 수집된 뉴스가 없습니다. 다른 날짜를 선택해보세요." 형태의 친절한 콜투액션(CTA) 버튼 유도 구현.
*   **🛡️ AI 태그 글자 폭주(Hallucination) UI 방어 (FE 관점):**
    *   백엔드에서 클렌징을 하더라도 예기치 못하게 비정상적으로 긴 태그 문자열이 넘어올 것에 대비해, 프론트엔드 태그 칩 컴포넌트에는 레이아웃 붕괴를 막는 CSS를 필수로 씌웁니다.
    *   적용 스타일: `max-width: 150px; white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`
