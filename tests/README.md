# Tests Directory User Guide

이 문서는 `tests/` 폴더에 존재하는 각종 테스트 및 검증 스크립트들의 **용도와 실행 방법**을 안내하는 매뉴얼입니다.
새로운 단위 테스트나 결함 검증 스크립트가 추가될 때마다 이 문서를 함께 업데이트합니다.

## 📌 테스트 스크립트 목록

| 파일명 | 용도 및 검증 대상 | 언제 사용하나요? (Use Cases) |
| --- | --- | --- |
| `test_backend_infra.py` | 1. DB 고유키 중복 방지 기능<br>2. `fcntl` 파일 락 좀비 프로세스 차단<br>3. 임시 데이터 DB 입출력 테스트 | 백엔드 인프라(DB, 스케줄러 락) 설정 변경 후, 시스템이 안정적으로 방어하는지 확인할 때. |
| `poc_rss_parsing.py` | RSS 피드 외부 요청 및 태그 파싱 구조 검증 | 크롤러 개발/수정 전, 특정 URL 피드가 정상적으로 읽히는지 단일 검증할 때. |
| `poc_ollama_call.py` | 로컬 Ollama API 호출(예: `gemma3:4b`) 연동 및 타임아웃/형식 검증 | AI 프롬프트를 변경하거나 모델을 교체했을 때 단독으로 AI 응답을 실험할 때. |
| `poc_telegram_msg.py` | Telegram Bot API 메시지 전송 및 권한 검증 | `.env`의 봇 토큰과 채팅 ID가 정상 작동하는지 외부 인터넷망 연동 테스트할 때. |
| `poc_schedule_loop.py` | OS 크론/스케줄러의 파이썬 기반 루프 및 시간 측정 검증 | 시간 스케줄링 로직 변경 후, 정해진 시간에 정확히 트리거되는지 실험할 때. |

---

## 🚀 실행 방법 가이드

모든 테스트 스크립트는 터미널에서 프로젝트 루트(`M-O-3-S.github.io/`) 위치에 있는 상태로 아래 명령어를 입력하여 개별 실행할 수 있습니다.

### 1. 백엔드 인프라(DB/Locking) 테스트
```bash
python3 tests/test_backend_infra.py
```
> **정상 결과 확인법:** 출력 마지막 줄에 `All Tests Passed!` 문구가 나타나며, 중간에 "Lock 'B' acquired!" 같은 에러 로그가 없어야 합니다.
