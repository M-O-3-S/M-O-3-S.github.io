# AI News Archive: 통합 테스트 케이스 (Test Cases)

본 문서는 기획자(사라)의 요구에 따라, 본 개발(Step 5)에 착수하기 전 기획팀(PRD, UI 가이드라인) 및 아키텍처 설계의 모든 요건을 충족하는지 검증하기 위해 작성된 **골든 스탠다드 로직 검증서 (Test Case)** 입니다. 

향후 기획(PRD)이나 UI 변경 없이 코드 리팩토링 및 구조 변경만 일어날 경우, 아래의 모든 TC를 100% 통과(Pass)해야만 코드를 병합(Merge)하고 배포할 수 있습니다. (회귀 테스트용)

## 1. 백엔드 & 파이프라인 (Backend & Data Pipeline) - 담당: 데이비드

| TC ID | 분류 | 테스트 명칭 (Scenario) | 사전 조건 (Pre-condition) | 기대 결과 (Expected Result) | 달성 여부 |
|-------|------|--------------------------|---------------------------|-----------------------------|:---:|
| BE-01 | 크롤러 | **정상 RSS 수집 블록 (Happy Path)** | `config.yml`에 유효한 RSS URL이 등록되어 있음 | 설정된 피드에서 XML을 파싱하여 제목, 링크, 발행일 데이터를 정상 추출한다. | [ ] |
| BE-02 | 크롤러 | **비정상 URL 방어 (Fallback)** | 1개의 RSS URL이 404, 307 등의 에러를 반환함 | 해당 URL만 `Timeout` 또는 오류 로깅 후 `Skip`하며, 전체 파이프라인이 중단(Crash)되지 않는다. | [ ] |
| BE-03 | 크롤러 | **방화벽 우회 (User-Agent)** | Arxiv 등 파이썬 기본 봇을 튕겨내는 URL 등록 | User-Agent 변조를 통해 차단당사지 않고 데이터를 긁어올 수 있다. | [ ] |
| BE-04 | DB저장| **중복 기사 필터링 (Deduplication)** | 이미 어제 수집된 뉴스 A가 오늘 다시 피드 최상단에 올라옴 | URL 해시 또는 제목+발행일을 대조하여 DB에 1건만 유지되고 중복 저장되지 않는다. | [ ] |
| BE-05 | 스케줄러| **OS-Agnostic 백그라운드 구동** | Windows, Linux(또는 Mac) 백그라운드에서 시스템 실행 | `cron` 없이 파이썬 예약 모듈만으로 기재된 시간(07:00, 토 08:00)에 정확히 파이프라인 전체가 발동된다. | [ ] |
| BE-06 | DB최적화| **데이터 보존 (Retention) 무결성** | 1년 치 이상의 더미 DB(JSON)가 적재됨 | 설정된 기한(예: 1년 외) 이후 오래된 DB 엔트리만 지워지나, JSON 파일들은 절대 지워지지 않고 영구 보존된다. | [ ] |
| BE-07 | 크롤러 | **초기 아카이브 벌크 수집 (Cold Start)** | DB 파일(`news_archive.db`)이 완전히 없는 완전 초기(Init) 상태 빈도 구동 | 오늘치 1~2개뿐만 아니라 과거 최대 N개/M일치의 피드 기사를 한 번에 전부 긁어와서 초기 데이터베이스를 채운다. | [ ] |

## 2. AI 모델 및 데이터 후처리 (LLM & Cleansing) - 담당: 에밀리

| TC ID | 분류 | 테스트 명칭 (Scenario) | 사전 조건 (Pre-condition) | 기대 결과 (Expected Result) | 달성 여부 |
|-------|------|--------------------------|---------------------------|-----------------------------|:---:|
| AI-01 | 추론 | **Model-Agnostic 설정 변환** | `config.yml`의 `llm_model` 값을 존재하지 않는 모델로 변경 | 파이썬 코드 수정 없이, 백엔드는 즉각 해당 모델명으로 API를 찔렀으나 '모델 없음' 에러를 내뱉음. | [ ] |
| AI-02 | 정제 | **영문 뉴스 한국어 요약 강제** | 수집된 해외 기사 원문(영문)이 프롬프트로 전달됨 | 영문이 일절 섞이지 않고 100% 한국어로 번역 및 최대 300자로 요약된 JSON을 반환한다. | [ ] |
| AI-03 | 정제 | **마크다운 찌꺼기 삭제 (Cleansing)** | Ollama 응답값에 ````json` 이나 불필요한 서술어가 포함됨 | 백엔드 후처리 정규식을 통해 순수한 `{ }` JSON 딕셔너리 포맷만 깔끔하게 추출되어 DB에 저장된다. | [ ] |
| AI-04 | 태그 | **태그 길이 및 갯수 규약 위반 복구** | 모델이 4개의 태그를 뱉거나, 빈 칸이 포함된 15자 이상의 태그를 뱉음 | 1회 Retry 발동 후, 연속 실패 시 시스템이 카테고리명을 기본 태그(1개)로 임시 할당하여 JSON 터짐을 방어한다. | [ ] |
| AI-05 | 태그 | **주간 베스트(Weekly) 빈도 집계** | 일주일 치 JSON이 취합되어 빈도 계산기를 거침 | 카테고리별로 태그 노출 빈도를 가장 정확하게 합산하여 상위 3개의 기사를 선출한다 (동점 시 최신순). | [ ] |

|-------|------|--------------------------|---------------------------|-----------------------------|:---:|

| FE-05 | 렌더링| **UI Empty State & 예외 처리** | 백엔드 에러로 텅 빈 JSON 파일이 떨어지거나, 로딩 중 통신 속도가 늦음 | 에러창(404) 대신 부드러운 Shimmer 애니메이션이 노출되거나 "수집된 뉴스가 없습니다" 등 친절한 그래픽이 렌더링된다. | [ ] |
