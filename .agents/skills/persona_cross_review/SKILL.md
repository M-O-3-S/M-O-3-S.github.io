# Skill: 페르소나 기반 크로스 문서 리뷰 (Pre-Flight Check)

## 목적
코딩 착수 **전에** 프로젝트 내 여러 문서(기획, 설계, UX, 요구사항 등) 간의 **불일치·모순·누락**을 찾아 선제적으로 수정한다.

다음 두 가지 낭비를 방지한다:
1. 개발 중간 발견되는 기획 모순 → 이미 짠 코드를 갈아엎는 최악의 사태
2. LLM이 상충하는 문서를 동시에 참조하며 발생하는 환각(Hallucination)

---

## 적용 조건
다음 중 하나라도 해당하면 코딩 전 반드시 먼저 실행한다:
- 신규 프로젝트 착수 직전
- 스테이크홀더가 2명 이상이고 각자 문서를 별도 작성한 경우
- 기획(PRD), 설계(Architecture), UX 문서 중 2종류 이상이 존재하는 경우
- 기능 요구사항이 3개 이상인 경우

---

## 실행 절차 (5단계)

### Step 1. 리뷰 대상 문서 목록 수집
리뷰할 모든 문서를 먼저 목록화한다.

```
- PRD / 요구사항 문서
- 아키텍처 설계 문서
- UX/디자인 명세서
- 개발 요구사항 (DEV_Requirements)
- 운영 매뉴얼 / 시스템 매뉴얼
- 설정 파일 (config.yml, .env.example 등)
```

### Step 2. 페르소나 배정
프로젝트의 실제 역할(스테이크홀더)에 맞게 리뷰어 페르소나를 설정한다.
각 페르소나는 **자신의 관심사 시각**으로만 문서를 읽고 충돌을 탐색한다.

| 페르소나 | 탐색 관점 |
| :--- | :--- |
| **아키텍트** | 기술 스택 일관성, 설정 키 이름, 타임아웃·용량 수치 |
| **백엔드 개발자** | DB 칼럼명·설정키 일치, 파일 경로 정합성 |
| **프론트엔드 개발자** | JSON 스키마 필드명, URL/파일 경로 |
| **QA/테스터** | 요구사항 시나리오 대비 테스트 케이스 누락 여부 |
| **PM(기획자)** | 기능 요구사항 대비 설계 문서의 기능 누락 여부 |
| **UX 디자이너** | 디자인 명세와 실제 컴포넌트 명세 불일치 |

> 프로젝트에 없는 역할은 건너뛰고, 해당하는 페르소나만 적용한다.

### Step 3. 교차 검증 (5가지 충돌 유형 체크)

| # | 유형 | 설명 | 예시 |
|---|------|------|------|
| 1 | **이름 불일치** | 같은 대상을 문서마다 다른 이름으로 부름 | `daily_time` vs `schedule.daily` |
| 2 | **수치 불일치** | 타임아웃·용량 등 숫자가 문서마다 다름 | DEV_Req: 10초 / Architecture: 5초 |
| 3 | **기능 누락** | 요구사항에 있지만 설계·테스트에 반영 안 됨 | PRD의 "초기 일괄 수집" 기능 미반영 |
| 4 | **경로/링크 불일치** | 참조 파일 경로가 실제 구조와 불일치 | `Doc/SWDesign/` → 실제는 `docs/SWDesign/` |
| 5 | **코드 결함** | 문서 내 코드 예시에 버그·중복 존재 | `except` 블록 중복 등 |

### Step 4. 결함 목록 작성 (Issue Log)
발견된 문제는 즉시 `cross_document_review.md` 아티팩트에 기록한다.

```markdown
## 발견된 결함 목록

| # | 심각도 | 유형 | 원인 문서 | 충돌 문서 | 내용 | 수정 방향 |
|---|------|------|---------|---------|-----|---------|
| 1 | Critical | 중복 코드 | PoC.md | - | except 블록 중복 | 중복 제거 |
| 2 | Major | 이름 불일치 | DEV_Req.md | config.yml | daily_time vs schedule.daily | DEV_Req 수정 |
```

- **Critical:** 코딩 착수 전 반드시 수정
- **Major:** 착수 전 수정 강력 권고
- **Minor:** 코딩 중 병행 수정 가능

### Step 5. 검증 완료 선언 후 코딩 착수
모든 Critical 이슈 수정 완료 후 "Clean State" 선언 → 코딩 착수 허용.

---

## 산출물 (Output)
- `cross_document_review.md` : 결함 목록 및 수정 내역 전체 기록
- 교차 참조 경로가 모두 정합성 확인된 문서 상태
- 코딩 착수를 위한 "Green Light" 상태

---

## 이 스킬이 처음 적용된 프로젝트
- **프로젝트:** AI News Archive (`M-O-3-S.github.io`)
- **적용 시점:** 아키텍처 설계 완료 후, 백엔드 코딩 착수 전
- **발견된 결함:** 12건 (Critical 4건, Major 4건, Minor 4건) 전량 수정 완료
- **주요 차단 버그:** `config.yml` 키 이름 불일치, 타임아웃 수치 불일치, PoC 중복 코드
